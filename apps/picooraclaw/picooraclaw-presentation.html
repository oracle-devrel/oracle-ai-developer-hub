Why Oracle AI Database?

  • In-database ONNX embeddings : vector generation without external services
  • AI Vector Search : native COSINE similarity on 384-dim vectors
  • ACID transactions : real durability for agent memory and state
  • Multi-agent isolation : session/workspace separation at the DB level
  • Graceful fallback : no Oracle? Falls back to local file-based storage

AI Vector Search

-- 384-dimensional ALL_MINILM_L12_V2 model
-- Loaded via in-database ONNX runtime

SELECT content, category,
  VECTOR_DISTANCE(embedding,
    VECTOR_EMBEDDING(ALL_MINILM_L12_V2
      USING :query_text),
    COSINE) AS distance
FROM PICO_MEMORIES
WHERE workspace_id = :ws
ORDER BY distance
FETCH FIRST 10 ROWS ONLY;

-- >30% similarity threshold applied in Go

Interface-Driven Architecture

type Tool interface {
  Name() string
  Description() string
  Execute(ctx context.Context, args map[string]any) (ToolResult, error)
}

type Channel interface {
  Name() string
  Listen(ctx context.Context) <-chan IncomingMessage
  Send(ctx context.Context, msg OutgoingMessage) error
}

type LLMProvider interface {
  Complete(ctx context.Context, req CompletionRequest) (CompletionResponse, error)
}

What Makes It Unique

  • Hardware I2C/SPI tools : the AI reads sensors, drives motors, talks to peripherals
  • MaixCAM vision channel : camera input as a first-class conversation channel
  • Workspace embedding : go:embed bakes default configs into the binary
  • Oracle graceful fallback : Oracle DB when available, local files when not
  • Pure Go Oracle driver : go-ora v2.9.0, no CGO, no InstantClient dependency
  • ~10MB footprint : runs on devices where Python can't even install
