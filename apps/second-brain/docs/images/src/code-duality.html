One table, two interfaces
Declare once how rows compose into a document. Read or write either shape; the engine keeps them the same rows.

CREATE TABLE posts (
  post_id       NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  platform_id   VARCHAR2(20) NOT NULL,
  kind          VARCHAR2(20) DEFAULT 'post',     -- post | reel | video | article
  title         VARCHAR2(1000),
  caption       CLOB,
  content_embedding VECTOR(384, FLOAT32)         -- semantic search, in-DB
);

-- one "post" served as a document, platform + media nested, fully read/write:
CREATE OR REPLACE JSON RELATIONAL DUALITY VIEW post_dv AS
  posts @insert @update @delete {
    _id : post_id   title : title   caption : caption   kind : kind
    platform : platforms { name : display_name }
    media    : media [ { url : url  kind : kind } ]
  };
read the view: one JSON document ·  write JSON through it: the engine updates the normalized rows  ·  no ORM, no sync code
