The embedding model lives inside the database
One call registers an open-source ONNX model in the engine. After this, embeddings are just SQL: no API, no key, no data leaving.
oracle/setup/01_load_onnx_model.sql
Step 1's bootstrap ran this for you
EXEC DBMS_VECTOR.LOAD_ONNX_MODEL(
  directory  => 'VEC_MODELS',                 -- where the .onnx file was mounted
  file_name  => 'all_MiniLM_L12_v2.onnx',    -- open-source MiniLM, 384 dims
  model_name => 'MINILM',                     -- the name SQL will call it by
  metadata   => JSON('{"function":"embedding", "embeddingOutput":"embedding",
                       "input":{"input":["DATA"]}}'));
from now on: VECTOR_EMBEDDING(MINILM USING :text AS DATA) anywhere SQL runs
